# Objection Handling

Map real objections to page sections. Do not create FAQ filler.

| Objection | Section pattern | Copy angle |
|---|---|---|
| Is this for me? | Audience fit / use cases | Name the exact user and situation |
| Can I trust you? | Proof / process / credentials | Show real proof or clear process |
| Is it expensive? | Value framing / comparison | Compare cost of inaction/alternatives |
| What happens next? | Process steps | Explain timeline and handoff |
| What if it fails? | Guarantee / risk reversal / FAQ | State realistic protection |
| Is it complicated? | Simple form / concierge path | Reduce effort and uncertainty |
| Why not competitor? | Comparison table | Be specific, not hostile |

## Rule

If there is no real answer to an objection, ask the user for business input instead of inventing one.
